# Step 6: Agent Workflow 생성 - 생산량 시뮬레이션

## 이 단계에서 무엇을 하는가

두 번째 Workflow를 생성합니다. `설비별 생산량 계산` Knowledge에 정의된 공식을 기반으로, 설비별 생산량을 시뮬레이션하고 납기 위험을 사전에 예측하는 Agent를 만듭니다.

Step 5의 "대기현황 모니터링"이 현재 상태를 보는 것이라면, 이 Workflow는 **미래를 예측**합니다.

---

## Workflow: 생산량 시뮬레이션

### 1. 어떤 메뉴에 들어가야 하는가

COS 좌측 메뉴 > Agents > 우측 상단 `+ New Agent` > `Create Workflow`

### 2. GitHub 파일 중 무엇을 참고하는가

- `knowledge/production-formula.md` - 핵심 공식 3개:
  - 일일 처리량 = monthly_throughput / 22
  - 예상 처리 일수 = queue_count / 일일 처리량
  - 납기 위험 = 예상 처리 일수 > 남은 영업일
- `csv/columns.md` - machine.csv의 monthly_throughput, queue_count 컬럼

### 3. 어떻게 단계별로 수행하는가

**3-1. Workflow 기본 설정**

1. `+ New Agent` 클릭
2. 아래 정보를 입력합니다:

| 항목 | 값 |
|------|-----|
| Workflow 이름 | `생산량 시뮬레이션` |
| Workflow 설명 | `설비별 일일 처리량을 계산하고, 현재 대기 주문의 예상 처리 일수와 납기 위험을 시뮬레이션한다.` |

**3-2. 에이전트 설정 - 온톨로지 객체 + 지식 매칭**

| 설정 항목 | 선택할 값 | 이유 |
|----------|----------|------|
| 연결 Object | `machine`, `print-order`, `gang-group` | 설비 성능, 주문 현황, 합대 현황 모두 필요 |
| 연결 Knowledge | `설비별 생산량 계산`, `합대 매칭 규칙` | 계산 공식 + 합대 규칙 참조 |

**3-3. Node 1 생성 - 설비별 처리량 계산**

1. 워크플로우 캔버스에서 `+ Add Node` 클릭
2. 아래 정보를 입력합니다:

| 항목 | 값 |
|------|-----|
| Node 이름 | `설비별 처리량 계산` |
| Node instruction | 아래 텍스트를 복사붙여넣기 |

```
machine 테이블에서 모든 설비의 monthly_throughput과 queue_count를 조회하세요.

Knowledge '설비별 생산량 계산'의 공식을 적용하여 각 설비의:
1. 일일 처리량 = monthly_throughput / 22
2. 예상 처리 일수 = queue_count / 일일 처리량

을 계산하세요. 정비 중(status='정비')인 설비는 처리량 0으로 표시하세요.
```

3. `Save` 클릭

**3-4. Node 2 생성 - 납기 위험 판정**

1. 캔버스에서 `+ Add Node` 클릭
2. 아래 정보를 입력합니다:

| 항목 | 값 |
|------|-----|
| Node 이름 | `납기 위험 판정` |
| Node instruction | 아래 텍스트를 복사붙여넣기 |

```
print_order 테이블에서 status가 '대기'인 주문의 due_date를 조회하세요.

Knowledge '설비별 생산량 계산'의 납기 위험 판정 기준을 적용하여 각 주문을 분류하세요:
- 정상: 남은 영업일 > 예상 처리 일수 + 2일
- 주의: 남은 영업일 <= 예상 처리 일수 + 2일
- 위험: 남은 영업일 <= 예상 처리 일수
- 긴급: due_date - today <= 3일 AND status = 대기

설비별로 각 등급의 주문 건수를 집계하세요.
```

3. `Save` 클릭

**3-5. Node 3 생성 - 시뮬레이션 리포트**

1. 캔버스에서 `+ Add Node` 클릭
2. 아래 정보를 입력합니다:

| 항목 | 값 |
|------|-----|
| Node 이름 | `시뮬레이션 리포트` |
| Node instruction | 아래 텍스트를 복사붙여넣기 |

```
설비별 생산량 시뮬레이션 결과를 한국어 리포트로 생성하세요.

출력 형식:
1. [설비별 처리량 요약]
   테이블: 설비명 | 인쇄방식 | 월간처리량 | 일일처리량 | 현재대기 | 예상처리일수

2. [납기 위험 주문 분석]
   등급별(정상/주의/위험/긴급) 주문 건수 + 긴급/위험 주문 상세 목록(order_id, product_name, due_date, 남은일수)

3. [설비 재배정 제안]
   부하가 높은 설비에서 여유 있는 설비로 재배정 가능한 주문이 있다면 제안.
   재배정 기준: 동일 machine_type(매엽/윤전) 설비 중 예상처리일수가 가장 짧은 설비
```

3. `Save` 클릭

**3-6. Node 연결**

1. Node 1 (설비별 처리량 계산) -> Node 2 (납기 위험 판정) -> Node 3 (시뮬레이션 리포트)
2. 순서대로 화살표를 드래그하여 연결
3. 전체 워크플로우를 `Save` 클릭

**3-7. 테스트 실행**

1. 우측 상단 `Test` 버튼 클릭
2. 테스트 질문 입력 (아래 텍스트를 복사붙여넣기):

```
설비별 생산량을 시뮬레이션하고 납기 위험이 있는 주문을 알려줘
```

3. `Run` 클릭

### 4. 성공했음을 어떻게 아는가

Agent가 아래와 같은 내용을 포함한 리포트를 생성하면 성공입니다:

- 설비별 일일 처리량이 계산됨 (예: M3 C호기 = 약 82건/일)
- M4 D호기는 정비 중이므로 처리량 0건으로 표시
- 납기 위험 등급별 주문 분류가 정확함
- 긴급 등급 주문이 상세 목록으로 표시됨 (ORD-041, ORD-042 등)
- 부하가 높은 설비(예: M5)에서 여유 설비로의 재배정 제안이 포함됨
