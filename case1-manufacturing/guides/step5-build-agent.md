# Step 5: Agent Workflow 생성

## 이 단계에서 무엇을 하는가

Agent Workflow를 생성하여 "설비 대기현황 모니터링" 업무를 자동화합니다. 2개 노드로 구성된 간단한 워크플로우를 만들고, 테스트 질문으로 동작을 확인합니다.

---

## Workflow: 설비 대기현황 모니터링

### 1. 어떤 메뉴에 들어가야 하는가

COS 좌측 메뉴 > Agents > 우측 상단 `+ New Agent` > `Create Workflow`

### 2. GitHub 파일 중 무엇을 참고하는가

- `knowledge/production-formula.md` - 일일 처리량 및 납기 위험 판정 공식
- `csv/columns.md` - 조회할 컬럼의 의미

### 3. 어떻게 단계별로 수행하는가

**3-1. Workflow 기본 설정**

1. `+ New Agent` 클릭
2. Workflow 이름: `설비 대기현황 모니터링`
3. Workflow 설명: `설비별 대기 주문 현황을 조회하고 납기 위험 주문을 식별하여 리포트를 생성한다.`

**3-2. Node 1 생성 - 대기현황 조회**

1. 워크플로우 캔버스에서 `+ Add Node` 클릭
2. Node 이름: `대기현황 조회`
3. Node instruction (아래 텍스트를 복사붙여넣기):

```
print_order 테이블에서 status가 '대기'인 주문을 machine_id 기준으로 그룹화하여 설비별 대기 건수를 집계하세요. 각 설비의 queue_count와 비교하여 검증하세요.
```

4. 연결할 Object: `print-order`, `machine`
5. `Save` 클릭

**3-3. Node 2 생성 - 리포트 생성**

1. 캔버스에서 `+ Add Node` 클릭
2. Node 이름: `리포트 생성`
3. Node instruction (아래 텍스트를 복사붙여넣기):

```
설비별 대기 현황을 한국어 리포트로 생성하세요. 납기 위험 주문(due_date가 3일 이내)을 강조 표시하세요.
```

4. `Save` 클릭

**3-4. Node 연결**

1. Node 1 (대기현황 조회)에서 Node 2 (리포트 생성)로 화살표를 드래그하여 연결
2. 전체 워크플로우를 `Save` 클릭

**3-5. 테스트 실행**

1. 우측 상단 `Test` 버튼 클릭
2. 테스트 질문 입력 (아래 텍스트를 복사붙여넣기):

```
현재 전체 설비의 대기 현황을 알려줘
```

3. `Run` 클릭

### 4. 성공했음을 어떻게 아는가

Agent가 아래와 같은 내용을 포함한 리포트를 생성하면 성공입니다:

- 설비별(A호기~E호기) 대기 주문 건수
- 각 설비의 인쇄 방식 (매엽/윤전) 표시
- D호기(M4)는 정비 중이므로 대기 0건으로 표시
- 납기가 3일 이내인 대기 주문이 강조 표시 (ORD-011, ORD-012, ORD-041, ORD-042 등)

## 추가 테스트 질문 (선택)

워크플로우가 정상 동작하면 아래 질문도 시도해 보세요:

- `A호기에 대기 중인 주문은 몇 건이야?`
- `납기가 가장 급한 주문 3건을 알려줘`
- `윤전 설비 중 대기 건수가 가장 많은 설비는?`
